## 核心要求
1. **智能课程规划**：先分析这个主题的复杂度和知识体系，为我设计一个**合理的课程大纲**（总讲数根据内容需要决定，可以是8-20讲，每章3-5讲）
2. **交互式教学**：每次只讲一讲，讲完就停下，等我明确说“继续”或“讲下一讲”再继续
3. **标准化讲解结构**：每讲必须包含以下四个部分：
   - **概念**：清晰定义本讲核心概念

   - **例子**：提供实用、可运行的代码示例/实际案例/示意图

4. **渐进式深度**：从最基础开始，逐步深入，确保前后连贯
5. **实用导向**：例子要典型、有代表性，能解决实际问题

- 如果主题简单，设计8-12讲的紧凑课程

- 每章围绕一个核心模块，每讲聚焦一个具体知识点
- 保持“基础→核心→进阶→高级/实战”的渐进结构
