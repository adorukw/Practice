# AdoruWorld 项目文档

口袋妖怪火红/绿宝石风格的个人博客系统，采用 Vue 3 + Express + Prisma + SQLite 技术栈。

## 技术栈

| 层 | 技术 |
|---|------|
| 前端框架 | Vue 3 + Composition API + TypeScript |
| 构建工具 | Vite 8 |
| 样式 | Tailwind CSS 4 + 自定义像素风格 |
| 状态管理 | Pinia |
| 路由 | Vue Router 5 |
| 后端 | Express + TypeScript (tsx 运行时) |
| ORM | Prisma 5 |
| 数据库 | SQLite (嵌入式，零配置) |
| Markdown | marked + highlight.js |

## 快速开始

### 环境要求

- Node.js >= 18
- npm >= 9

### 1. 安装依赖

```bash
# 前端依赖
cd AdoruWorld
npm install

# 后端依赖
cd server
npm install
```

### 2. 初始化数据库

```bash
cd server

# 生成 Prisma Client
npm run db:generate

# 执行数据库迁移（创建表结构）
npm run db:migrate -- --name init

# 填充种子数据（分类、标签、项目等）
npm run db:seed
```

### 3. 开发启动

需要同时启动前端和后端两个服务：

**终端1 - 启动后端**

```bash
cd server
npm run dev
# Server running on http://localhost:3002
```

**终端2 - 启动前端**

```bash
cd AdoruWorld
npm run dev
# VITE ready → http://localhost:5173
```

> 前端通过 Vite proxy 将 `/api` 请求代理到后端 `http://localhost:3002`。

### 4. 访问

| 地址 | 说明 |
|------|------|
| http://localhost:5173 | 前台博客 |
| http://localhost:5173/admin | 管理后台（无需登录） |
| http://localhost:3002/api/health | 后端健康检查 |

## 数据库管理

```bash
cd server

# 可视化浏览/编辑数据库
npm run db:studio
# Prisma Studio → http://localhost:5555

# 重新填充种子数据（会覆盖现有数据）
npm run db:seed
```

## 管理后台

访问 `/admin` 可直接管理所有数据，无需账号登录（自用设计）。

支持管理的数据：
- **文章** — 标题、内容(Markdown)、封面图、分类、标签、发布状态、精选
- **分类** — 名称、图标、颜色
- **标签** — 名称、颜色
- **项目** — 名称、描述、技术栈、链接、状态
- **图鉴** — 影视/游戏/书籍/音乐记录、评分、进度

## 打包部署

### 前端构建

```bash
cd AdoruWorld
npm run build
# 输出目录: dist/
```

产物为纯静态文件，可部署到任意静态托管服务（Nginx、Vercel、Netlify 等）。

### 后端构建

```bash
cd server
npm run build
# 输出目录: dist/
npm run start
```

### Nginx 部署示例

```nginx
server {
    listen 80;
    server_name your-domain.com;

    # 前端静态文件
    root /path/to/AdoruWorld/dist;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    # API 代理到后端
    location /api/ {
        proxy_pass http://localhost:3002;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

生产环境中需要更新 `src/api/index.ts` 中的 `API_BASE` 为你的实际域名或空字符串（同域部署时）。

## 数据模型

### 文章 (posts)
| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| slug | String | URL 友好标识 |
| title | String | 标题 |
| description | String | 摘要 |
| content | String | Markdown 正文 |
| coverImage | String? | 封面图 URL |
| published | Boolean | 是否发布 |
| featured | Boolean | 是否精选 |
| categories | 多对多 | 关联分类 |
| tags | 多对多 | 关联标签 |

### 图鉴 (dex_entries)
| 字段 | 类型 | 说明 |
|------|------|------|
| title | String | 标题 |
| category | String | 分类 (anime/movie/tv/game/book/music) |
| status | String | 状态 (completed/watching/playing/reading/listening/dropped/planned) |
| rating | Int | 评分 0-10 |
| progress | Int | 当前进度 |
| total | Int | 总集数/章节数 |

## 常见问题

**Q: 端口被占用？**
```bash
# 查找并释放端口
fuser -k 3002/tcp
fuser -k 5173/tcp
```

**Q: 数据库出问题需要重置？**
```bash
cd server
rm -f prisma/dev.db
npx prisma migrate dev --name init
npm run db:seed
```

**Q: 如何修改后端端口？**
编辑 `server/src/index.ts` 中的 `PORT` 值，同时更新 `vite.config.ts` 的 proxy target。
