## 二、你的数据该怎么“建模”（这是关键）

你的 `messages[]` 不是扁平日志，而是 **多层结构**：

```
user
 └─ assistant
     ├─ thinking
     ├─ toolCall
     │    └─ toolResult
     ├─ thinking
     ├─ toolCall
     │    └─ toolResult
     └─ text
```

### 1️⃣ 定义统一 Node 模型（非常关键）

```ts
// types.ts
export interface SessionNode {
  id: string
  type:
    | 'user'
    | 'assistant'
    | 'thinking'
    | 'toolCall'
    | 'toolResult'
    | 'system'
  label: string
  raw: any
  parentId?: string
  meta?: {
    toolName?: string
    toolCallId?: string
    duration?: number
    error?: boolean
    cost?: number
    tokens?: number
  }
  timestamp: number
}
```

### 2️⃣ 从 session_info.json → nodes + edges

**转换逻辑（伪代码）**：

```ts
const nodes: SessionNode[] = []
const edges: Edge[] = []

messages.forEach((msg, index) => {
  if (msg.role === 'user') {
    nodes.push(userNode(msg))
  }

  if (msg.role === 'assistant') {
    msg.content.forEach(block => {
      if (block.type === 'thinking') {
        nodes.push(thinkingNode(block))
        edges.push(edge(parent, block))
      }
      if (block.type === 'toolCall') {
        nodes.push(toolCallNode(block))
        edges.push(edge(parent, block))
      }
      if (block.type === 'text') {
        nodes.push(textNode(block))
      }
    })
  }

  if (msg.role === 'toolResult') {
    nodes.push(toolResultNode(msg))
    edges.push({
      id: `${msg.toolCallId}->result`,
      source: msg.toolCallId,
      target: msg.id
    })
  }
})
```

✅ **这样你就得到了一个真正的 DAG，而不是“假时间线”**

## 三、Vue Flow 如何表达你的会话流（重点）

### 1️⃣ 节点长什么样（视觉设计）

| 类型       | 颜色    | 形状       |
| ---------- | ------- | ---------- |
| user       | 蓝      | 圆角矩形   |
| assistant  | 紫      | 胶囊       |
| thinking   | 灰      | 虚线卡片   |
| toolCall   | 橙      | 代码块样式 |
| toolResult | 绿 / 红 | 折叠面板   |
| error      | 红边框  | 闪烁提示   |

示例（Tailwind）：

```vue
<template>
  <div
    class="px-3 py-2 rounded-lg border text-sm"
    :class="{
      'bg-blue-50 border-blue-300': type === 'user',
      'bg-orange-50 border-orange-400': type === 'toolCall',
      'bg-red-50 border-red-400': error
    }"
  >
    {{ label }}
  </div>
</template>
```

---

### 2️⃣ 边的语义

- **顺序边**：消息先后顺序（细灰线）
- **调用边**：`toolCall → toolResult`（粗箭头）
- **错误边**：红色虚线

```ts
{
  id: 'call->result',
  source: 'call_xxx',
  target: 'result_xxx',
  animated: true,
  style: { stroke: '#f59e0b' }
}
```
